Custom KSounds In Neptune

A quick detailed guide showing how to set custom key sounds in Neptune works for all platforms.

Table of Contents
  • Choosing your custom keysounds.
  • Creating the custom key sound directory
  • Renaming sounds keys.
  • Sound Key events.
  • Creating a config.json file
  • Fallback Keys.
  • Playing the custom sound.

Choosing your custom keysounds.

When choosing a keysound for a key it should be either in wav or ogg (vorbis) format. For example I’ve downloaded a bunch of sounds from https://freesound.org

Some of them are waveform and some are not .wav

So I’ll have to convert those that are not to .wav or .ogg

Creating the custom key sound directory

Browse to Neptune directory based on the platform you’re using.

  • For Linux: ~/.local/share/Neptune
  • For Windows: %AppData%\Neptune (C:\Users\Ybenel\Local\Appdata\Roaming\Neptune)
  • For MacOs: Library\Application Support\Neptune

Now we’ll have to create a directory with the name of keysound. and now we’ll move our files there.

Renaming sounds keys.

Now as we moved our files. we’re going to have to rename them to the appropriate keys so Neptune can recognize them and play them when we hit that key. One of our sounds typewriter-ding_near_mono.wav when i want to hit enter I’d like to play this sound.

So what I’ll do is to just rename it to enter.wav.

typewriter-ding_near_mono.wav (Click enter key ) -> renamed to enter.wav

If you want to rename to a key on your keyboard but you don’t know its name. You can run the following command in Neptune and I’ll show you the key you pressed.

Neptune -cli -verbose

As you can see when i hit the enter key on my keyboard and it displayed keychar which is the key name that we will use.

Another way is to read this file and use the names presented after the code to name your files. Like left_arrow num_lock lbracket and so on.

Sound Key events.

After you have renamed all your keys. and you want to add a key event for each key pressed and released. Make sure that file is also in the same directory. and rename it to that key you already have named with -up. For Example.

You have backspace.wav this will be played when you press the key, And backspace-up.wav will be played when your release that key.

Creating a config.json file

When u read config.json you’d think we need to write configuration of each keys but that’s not true, we need to create it to help Neptune recognize that our Custom Folder is a SoundKey . However, keeping the file empty works for keysounds that don’t have release key-event.

If you have a key-event you just need to add this to the file.

{
  "name": "My Awesome SoundKey (Optional)",
  "support-events": true
}

Fallback Keys.

Fallback keys are keysounds that will be played for each key that doesn’t have its sound-key. it works sounds that support key-events or those that doesn’t. You can have as much fallback key-sounds as you want, as long as they have “fallback” key word on them. If you have multiple fallback keys that will be played randomly for each key.

fallback.wav
fallback2.wav
fallback3.ogg
fallbackgg.wav
fallback100.wav
fallback1000.wav
fallbacksss.wav

Playing the custom sound.
